## 0.1 — Module map

### `src/engine/` (the core; every function returns a `Result`, every side effect is injected)

| Module | Responsibility | Symbols a read-model/TUI will cite |
| --- | --- | --- |
| `git.ts` | Injectable wrapper over the `git` CLI; read + mutation primitives, never throws. | `createGit(repoRoot, run?, runRaw?)`, interface `Git` with `statusPorcelain()`, `changedFiles(base, branch)`, `currentBranch()`, `listWorktrees()`, `branchExists(name)`, `mergeTree(a,b)`; types `WorktreeInfo {path, head, branch?}`, `GitError`; pure `parseWorktreeList`, `normalizeRepoPath`. |

| `integrator.ts` | Inspects real git state pre-integration; renders the markdown report. | `analyzeIntegration(config, projectRoot, run?, runRaw?)` → `IntegrationAnalysis`; `integrationSummaryLine(analysis)` (ready compact status); report path `resolveBoardDir(...)/reports/integration.md`. | `worktrees.ts` | Ensures each session worktree + deps. | `ensureWorktrees(git, config, projectRoot)`; session→worktree path is `path.join(projectRoot, session.worktree)`; `EnsureAction 'created'\|'exists'`. |
| `branches.ts` | Ensures integration + per-session branches (base → integration → session). | `ensureIntegrationBranch(git, config)`, `ensureSessionBranches(git, config)`; exports the shared `EnsureAction`. |
| `database.ts` | Detects Postgres, creates per-session DB, degrades to manual command. | `detectDbSetup(projectRoot)`, `ensureSessionDatabases(config, projectRoot, run?)`; `SessionDbResult.action 'created'\|'exists'\|'skipped'\|'manual'`. |

### `src/mcp/`

- **`server.ts`** — `startRwMcpServer()` connects `createRwMcpServer(deps?)` to a **stdio** transport (per-MCP-session, single mutable `sessionToken`; not multi-client). Metadata `{name:'rw-ai', version: readVersion()}`. This is why §4.3 of the arch doc says the MCP server **cannot** double as the Phase-2 daemon.
- **`tools.ts`** — 10 pure handlers: `handleStatus`, `handleRoles`, `handleWhoami`, `handleClaim`, `handleRelease`, `handleCheck`, `handleFinish`, `handleLaneCheck`, `handleBootstrap`, `handleAddRemote`; `ToolDeps {now, run?, runRaw?, token?}`.
- **`context.ts`** — `resolveMcpContext(cwd, run?, runRaw?)` → `McpContext {sharedRoot, config, boardDir, currentSession?}`; resolves the shared root from `git worktree list --porcelain` (main worktree first).

### `src/cli/`

- **`run.ts`** — `runCli(argv, deps)` is the single testable entry. `route()` switch dispatches subcommands; `USAGE` const (single source of help text). `mcp` is dispatched by the bin entry, not `route()`.
- **`command.ts`** / **`context.ts`** — `CommandResult {lines, exitCode, stderr?}`, `CliDeps`; `loadContext(cwd,...)` → `CliContext {projectRoot, config, git}` (resolves via `git.toplevel()`).
- **`version.ts`** — `readVersion()` (memoized; shared by CLI `--version` and MCP metadata).
- **`commands/`** subcommands: `bootstrap, scaffold, configure, adapters, roles, init, claim, whoami, release, finish, add-session, archive, check, lane, lane-guard, session-start, sessions, tokens`.

## 0.2 — Git primitives for the read model

| Needed by read model | Status | Real signature |
| --- | --- | --- |
| `statusPorcelain` | ✅ exists | `statusPorcelain(): Promise<Result<string, GitError>>` (`git status --porcelain`). Working-tree dirty state. |
| `changedFiles` | ✅ exists | `changedFiles(base, branch): Promise<Result<string[], GitError>>` — **three-dot `diff --name-only base...branch` between refs**, not working-tree changes. |
| `currentBranch` | ✅ exists | `currentBranch(): Promise<Result<string, GitError>>` (`rev-parse --abbrev-ref HEAD`; returns literal `"HEAD"` when detached — no special handling). |
| `listWorktrees` | ✅ exists | `listWorktrees(): Promise<Result<WorktreeInfo[], GitError>>`; detached entries simply omit `branch`. |
| ahead/behind vs base | ❌ **GAP confirmed** | No `aheadBehind`, no `rev-list --count`, no `@{upstream}`. Genuinely absent — WU-1.3 is new work. |

**Injection pattern (for WU-1.3):** git primitives are methods on the object from
`createGit(repoRoot, run = runCommand, runRaw = runCommandRaw)` — two **positional**
`CommandRunner`s bound to `repoRoot`. `run` = strict (ok only on exit 0), `runRaw` =
non-zero exit is data. There is no `{ exec }` options bag anywhere.

## 0.3 — Version drift

- `package.json` `version` = **`0.1.0`**; milestone is **v0.5**. Drift confirmed.
- Recommendation: keep the plan's "v0.5" as the **milestone/epic label**, not a
  package version. Bumping `package.json` to `0.5.0` is a release decision, not a
  Phase-0 action — defer to the release step. `readVersion()` feeds both CLI
  `--version` and MCP server metadata, so any bump changes user-visible output and
  should be its own deliberate commit.

## Corrections the audit forces on later work units

These are the real payoff of Phase 0 — the plan text must be adjusted before coding:

1. **WU-1.1 (`.review-board/sessions/<id>/status.json`)** — `ensureBoard` today
   creates `screenshots/<sid>`, `test-artifacts/<sid>/<platform>`, `data`, `reports`,
   `tasks`. It does **not** create a `sessions/` subtree. WU-1.1/1.2 must **extend
   `ensureBoard`** to provision `sessions/<id>/`, reusing `writeFileIdempotent` +
   `writeFileAtomic` — not assume it exists.

2. **WU-1.3 signature** — replace the proposed
   `aheadBehind(worktreePath, baseBranch, { exec })` with a method on the `Git`
   interface: `aheadBehind(base: string): Promise<Result<{ahead, behind}, GitError>>`,
   obtained via `createGit(worktreePath, run, runRaw)` — matching `currentBranch()` /
   `statusPorcelain()`, which are already worktree-scoped by their bound `repoRoot`.
   No `{ exec }` bag exists in the codebase. The new method must define its own
   detached-HEAD / no-merge-base / no-upstream behavior — there is **no existing
   convention** to inherit (nothing in `git.ts` consults an upstream today).

3. **WU-1.4 "git dirty"** — the plan cites `git.statusPorcelain`/`changedFiles`
   together, but they answer different questions: **dirty working tree** →
   `statusPorcelain()`; **branch divergence from a ref** → `changedFiles(base, branch)`
   (three-dot). The light rules ("git dirty/ahead") must map to `statusPorcelain()`
   for dirty and the new `aheadBehind` for ahead — not `changedFiles`.

4. **WU-1.2 agent-agnostic marker** — the Claude-specific signal lives in
   `src/engine/claude-sessions.ts` (surfaced by the `sessions` CLI command via
   `commands/claude-sessions.ts`). Generalizing it into the marker means writing the
   marker from the lifecycle/`session-start` path, keeping `claude-sessions.ts` as one
   optional input, not the contract.

5. **Phase 2 daemon** — confirmed the MCP server is stdio + per-session single-token
   (`server.ts`); it is not reusable as the multi-client daemon (arch §4.3 / WU-2.0
   holds). `identity.listRoles`/`readClaims` are lock-free reads — the daemon can poll
   them safely.

6. **`rw status` CLI** — confirmed absent from `route()` and `USAGE`. Only the MCP
   tool `rw_status` exists (different surface). WU-1.5 adds the CLI command cleanly.
